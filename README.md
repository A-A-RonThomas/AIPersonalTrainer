# AI Personal Trainer

A weekly workout-programming service built on a Push/Pull/Legs skeleton. Generates a week of training, lets you review and edit it via chat, then pushes the approved plan to Hevy. The next week is generated by evaluating actual performance from the prior week.

## Prerequisites

- Python 3.11+
- PostgreSQL
- [Hevy Pro](https://hevy.com) account (API key from hevy.com/settings?developer)
- Anthropic API key

## Setup

### 1. Clone and create a virtual environment

```bash
git clone <repo-url>
cd personal-trainer
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\Activate.ps1
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Edit `.env` with your values:

```
DATABASE_URL=postgresql://user:password@localhost/personal_trainer
HEVY_API_KEY=your_hevy_api_key_here
ANTHROPIC_API_KEY=your_anthropic_api_key_here
```

### 4. Create the database and run migrations

```bash
createdb personal_trainer
alembic upgrade head
```

### 5. Run the app

```bash
uvicorn app.main:app --reload
```

The app runs at `http://localhost:8000`.

## Deployment (home server)

The app is designed to run as a systemd service on a home server behind Tailscale. A typical unit file:

```ini
[Unit]
Description=AI Personal Trainer
After=network.target postgresql.service

[Service]
WorkingDirectory=/path/to/personal-trainer
EnvironmentFile=/path/to/.env
ExecStart=/path/to/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

## Usage

1. Open the app in your browser
2. The weekly plan is generated each Monday (or on demand)
3. Review the plan — a diff vs last week is shown
4. Use the chat box to request edits ("swap barbell rows, shoulder is sore", "only 4 days this week")
5. Approve the plan — this is the only action that writes to Hevy

Nothing is pushed to Hevy until you explicitly approve.
